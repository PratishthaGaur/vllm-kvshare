    │                                     │
    ├─ Timestamp (arrival times)        ├─ Model
    ├─ Request tokens                   ├─ GPU config
    ├─ Response tokens                  └─ vLLM settings
    └─ Model type (Chat/GPT-4)


                          ↓ BENCHMARK RUNS ↓

PROCESSING
──────────
For each request:
  1. Read from trace (timestamp, token counts)
  2. Generate prompt with exact token count
  3. Wait for inter-arrival time
  4. Send to vLLM server
  5. Collect response metrics

HOW TO ACCESS
─────────────

View summary in console:
  $ tail -50 results/benchmark.log

Python analysis:
  >>> import json
  >>> with open('results/results.json') as f:
  ...     data = json.load(f)
  >>> data['metrics'][0]
  {...}

Spreadsheet import:
  $ open results/results.csv

Comparison:
  $ python analyze_burstgpt_results.py \
      --compare results/baseline results/optimized


KEY METRICS AT A GLANCE
───────────────────────

For SLA compliance:
  → Watch P99 latency (should be < 2s)

For user experience:
  → Watch P99 time-to-first-token (should be < 500ms)

For system health:
  → Watch success rate (should be > 95%)

For efficiency:
  → Watch output tokens/sec (should be > 100)


WORKFLOW
────────

1. Run benchmark:
   $ python burstgpt_trace_replay.py --model llama-2-7b-hf

2. Get summary:
   $ tail -50 results/benchmark.log

3. Detailed analysis:
   $ python analyze_burstgpt_results.py --results-dir results/benchmark.log

4. Compare configurations:
   $ python analyze_burstgpt_results.py --compare results/config_a results/config_b
