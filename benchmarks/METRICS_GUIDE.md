# Comprehensive Metrics Guide

Complete breakdown of all data and metrics collected by the BurstGPT trace replay benchmark.

## Overview

The benchmark collects data at three levels:

1. **Per-Request Metrics** (per individual request)
2. **Aggregate Statistics** (computed from all requests)
3. **Configuration & Metadata** (benchmark settings)

---

## 1. Per-Request Metrics

Each request generates a detailed record with the following fields:

### Basic Request Information

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `request_id` | int | Sequential request number (0-indexed) | `0`, `1`, `523` |
| `timestamp` | float | Scheduled request time from trace (seconds) | `5.23`, `45.67` |
| `request_tokens` | int | Input token count from trace | `472`, `1087` |
| `expected_output_tokens` | int | Expected output tokens from trace | `18`, `230` |

### Actual Request Results

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `actual_input_tokens` | int | Actual tokens sent (after padding/truncation) | `472`, `1087` |
| `actual_output_tokens` | int | Actual tokens generated by model | `18`, `230` |
| `time_to_first_token` | float | Latency to first token (seconds) | `0.095`, `0.156` |
| `total_latency` | float | Total response time (seconds) | `0.384`, `0.521` |
| `error` | string or null | Error message if request failed | `null`, `"HTTP 500: ..."` |

### Example Request Entry (JSON)

```json
{
  "request_id": 42,
  "timestamp": 234.5,
  "request_tokens": 472,
  "expected_output_tokens": 18,
  "actual_input_tokens": 472,
  "actual_output_tokens": 18,
  "time_to_first_token": 0.142,
  "total_latency": 0.384,
  "error": null
}
```

## 2. Aggregate Statistics

Computed from all requests and printed to logs + saved in results.

### Request Counts

```
Total requests: 1000
Successful: 980
Failed: 20
Success rate: 98.00%
```

| Metric | Description | Use Case |
|--------|-------------|----------|
| **Total requests** | Number of requests sent | Understanding workload size |
| **Successful** | Requests that completed without error | Health check |
| **Failed** | Requests that failed | Error analysis |
| **Success rate** | % of successful requests | SLA compliance (target: >95%) |

### Latency Metrics - Total Response Time

```
Mean:      0.486s
Median:    0.412s
P95:       0.856s
P99:       1.234s
```

| Metric | Description | Typical Target |
|--------|-------------|-----------------|
| **Mean** | Average latency across all requests | <500ms for LLM serving |
| **Median** | Middle value (50th percentile) | <400ms |
| **P95** | 95th percentile (5% slower) | <1000ms |
| **P99** | 99th percentile (1% slowest) | <2000ms |

**Use case**: Understanding end-to-end latency distribution. P99 is critical for SLAs.

### Latency Metrics - Time to First Token (TTFT)

```
Mean:      95.23ms
Median:    87.45ms
P95:       234.56ms
P99:       456.78ms
```

| Metric | Description | Typical Target |
|--------|-------------|-----------------|
| **Mean** | Average prefill latency | <100ms for interactive apps |
| **Median** | Middle prefill latency | <90ms |
| **P95** | 95th percentile prefill latency | <250ms |
| **P99** | 99th percentile prefill latency | <500ms |

**Use case**: Measuring user-perceived responsiveness. Critical for interactive chat applications.

### Throughput Metrics

```
Requests/sec: 4.12
```

| Metric | Description | Calculation | Typical Value |
|--------|-------------|-------------|----------------|
| **Requests/sec** | Request throughput | successful_requests / total_time | 2-10 req/s |

### Token Metrics

```
Total input tokens:  485,230
Total output tokens: 98,450
Input tokens/sec:    2,047.88
Output tokens/sec:   415.49
```

| Metric | Description | Calculation | Typical Value |
|--------|-------------|-------------|----------------|
| **Total input tokens** | All prompt tokens processed | Sum of input tokens | 100k-1M |
| **Total output tokens** | All generated tokens | Sum of output tokens | 10k-500k |
| **Input tokens/sec** | Prompt processing throughput | total_input / total_time | 1000-5000 tok/s |
| **Output tokens/sec** | Generation throughput | total_output / total_time | 100-1000 tok/s |

### Benchmark Duration

```
Total benchmark time: 243.52s
```

The wall-clock time to complete the entire benchmark run.

## 3. Configuration & Metadata

Saved in results.json under `benchmark_config`:

```json
{
  "benchmark_config": {
    "trace_path": "data/BurstGPT/data/BurstGPT_1.csv",
    "model": "llama-2-7b-hf",
    "base_url": "http://localhost:8000",
    "scale": 1.0,
    "streaming": false,
    "timestamp": "2024-03-22T10:30:15.234567"
  }
}
```

| Field | Description | Example |
|-------|-------------|---------|
| `trace_path` | Path to the trace file used | `data/BurstGPT/data/BurstGPT_1.csv` |
| `model` | Model name | `llama-2-7b-hf` |
| `base_url` | vLLM server URL | `http://localhost:8000` |
| `scale` | Trace replay speed factor | `1.0` (real-time), `10.0` (10x faster) |
| `streaming` | Whether streaming mode was used | `true` or `false` |
| `timestamp` | When benchmark was run | `2024-03-22T10:30:15.234567` |

## 5. Error Tracking

### Error Breakdown

When failures occur, the log shows:

```
Error breakdown:
  HTTP 503: Service Unavailable: 5
  Request timeout after 300s: 3
  Failed to connect to server: 12
```

| Error Type | Cause | Action |
|-----------|-------|--------|
| `HTTP 503` | Server overloaded | Reduce concurrency or scale up |
| `HTTP 500` | Server error | Check server logs |
| `timeout` | Request took too long | Increase `--timeout` parameter |
| `ConnectionError` | Server unreachable | Verify server is running |

## 6. Statistical Aggregates Computed

### Latency Distribution

For each successful request's latency:
- Calculates: mean, median, P95, P99, min, max
- Uses: numpy.percentile()
- Applies to: total_latency, time_to_first_token

### Token Statistics

For all successful requests:
- Sum of actual_input_tokens
- Sum of actual_output_tokens
- Rate = tokens / total_time

### Success Metrics

```python
success_rate = successful_requests / total_requests
failed_requests = total_requests - successful_requests
```

## 7. How to Access Each Metric

### From Logs (Real-time)

```bash
tail -f results/benchmark.log
```

Shows all metrics as they're computed.

---

### From JSON (Programmatic Access)

```python
import json

with open('results/results.json') as f:
    data = json.load(f)

# Access config
model = data['benchmark_config']['model']

# Access summary
success_rate = data['summary']['successful_requests'] / data['summary']['total_requests']

# Access per-request metrics
for metric in data['metrics']:
    print(f"Request {metric['request_id']}: {metric['total_latency']:.3f}s")
```

---

### From CSV (Spreadsheet Analysis)

```bash
# Open in Excel, Sheets, or Python

df = pd.read_csv('results/results.csv')

# Group by request size
by_size = successful.groupby('request_tokens')['total_latency'].mean()
```

## 9. Data Quality Checks

When analyzing results, verify:

✅ **Token Matching**: `expected_output_tokens ≈ actual_output_tokens`
- Should be very close or identical
- If not, check tokenizer compatibility

✅ **Request Counts**: `successful + failed = total`
- Always true by definition
- If not, data corruption occurred

✅ **Latency Ordering**: `time_to_first_token ≤ total_latency`
- Always true (TTFT is part of total)
- If violated, data error

✅ **No Missing Data**: `error is null OR error is string`
- Every request has outcome
- If missing, benchmark incomplete

## 10. Example Analysis Script

```python
import json
import numpy as np
from pathlib import Path

results_dir = Path('results/my_benchmark')

# Load data
with open(results_dir / 'results.json') as f:
    data = json.load(f)

metrics = data['metrics']
successful = [m for m in metrics if m['error'] is None]

print(f"Total Requests: {len(metrics)}")
print(f"Success Rate: {len(successful)/len(metrics)*100:.1f}%")

# Latency analysis
latencies = [m['total_latency'] for m in successful]
print(f"\nLatency (seconds):")
print(f"  Mean:   {np.mean(latencies):.3f}")
print(f"  Median: {np.median(latencies):.3f}")
print(f"  P99:    {np.percentile(latencies, 99):.3f}")

# Token analysis
input_tokens = sum(m['actual_input_tokens'] for m in successful)
output_tokens = sum(m['actual_output_tokens'] for m in successful)
total_time = max(m['timestamp'] for m in successful)

print(f"\nToken Statistics:")
print(f"  Input:  {input_tokens:,} tokens ({input_tokens/total_time:.0f} tok/s)")
print(f"  Output: {output_tokens:,} tokens ({output_tokens/total_time:.0f} tok/s)")

# Error analysis
errors = [m for m in metrics if m['error'] is not None]
if errors:
    print(f"\nErrors ({len(errors)}):")
    error_types = {}
    for m in errors:
        error_type = m['error'].split(':')[0]
        error_types[error_type] = error_types.get(error_type, 0) + 1
    for err_type, count in sorted(error_types.items(), key=lambda x: -x[1]):
        print(f"  {err_type}: {count}")
```

## 11. Comparison Metrics

When using `analyze_burstgpt_results.py --compare`:

```
Comparison vs Baseline: baseline
```

Speedup calculations:
- `1.05x` = 5% slower (5% more latency)
- `0.95x` = 5% faster (5% less latency)
